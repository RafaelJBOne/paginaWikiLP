Wiki: Projeto P2P GoLang

1. Introdução Geral

Este projeto consiste em um sistema de compartilhamento de arquivos ponto-a-ponto (P2P) com um rastreador (tracker) centralizado. Desenvolvido inteiramente na linguagem Go, o sistema permite que múltiplos "peers" se conectem para compartilhar, buscar e baixar arquivos entre si. Além da transferência de arquivos, o projeto suporta a troca de mensagens de chat em tempo real e oferece duas formas de interação: uma interface de linha de comando (CLI) e uma interface web acessível via navegador.

2. Por que Go? (Concorrência e Paralelismo)

A escolha da linguagem Go é fundamental para este projeto devido ao seu modelo nativo e eficiente de concorrência, que é explorado em funcionalidades críticas:

  • Downloads Paralelos Multi-peer: O sistema utiliza um pool de goroutines para baixar diferentes "chunks" (pedaços) de um arquivo de múltiplos peers simultaneamente. Isso permite que a largura de banda seja otimizada ao solicitar partes distintas do arquivo para diferentes fontes ao mesmo tempo.
  • Escrita Não-Bloqueante: Graças ao uso da função WriteAt, cada goroutine escreve seu pedaço diretamente na posição correta do arquivo de destino. Como os intervalos de bytes não se sobrepõem, o projeto consegue paralelismo real na escrita sem a necessidade de travas (locks) complexas.
  • Servidores Simultâneos: Tanto o Peer quanto o Tracker utilizam goroutines para gerenciar múltiplas conexões TCP de entrada sem travar a execução principal. No Peer, isso permite que ele envie um arquivo para alguém enquanto recebe uma mensagem de chat ou baixa outro arquivo.

2.1 Comparação Go em relação à linguagem C

Embora a linguagem C ofereça alto desempenho e amplo controle sobre os recursos do sistema, o desenvolvimento de uma aplicação P2P envolve desafios relacionados à concorrência, comunicação em rede e gerenciamento de memória, aspectos nos quais Go oferece recursos nativos que reduzem a complexidade do código e aumentam sua confiabilidade.

Premissas de usuário e domínio de aplicação

Linguagem C

Para implementar uma aplicação peer-to-peer (P2P) em C, é necessário utilizar bibliotecas externas para concorrência e comunicação em rede, como pthreads e outras abstrações para sockets. Além disso, o gerenciamento de memória deve ser realizado manualmente pelo desenvolvedor.

Go

Go disponibiliza bibliotecas nativas como net, sync e encoding/json, permitindo o desenvolvimento completo da aplicação utilizando apenas a biblioteca padrão da linguagem, sem dependências externas.

Construtores da linguagem

Linguagem C

A linguagem C não possui mecanismos nativos equivalentes a goroutines, channels ou closures. A concorrência é normalmente implementada por meio de pthread_create, mutexes gerenciados manualmente e callbacks baseados em ponteiros para funções.

Go

Go oferece goroutines, channels, sync.Mutex e sync.WaitGroup como recursos nativos. Dessa forma, tarefas concorrentes, como o download paralelo de partes de arquivos entre peers, podem ser implementadas de maneira mais simples e com menos código.

Legibilidade e manutenção

Linguagem C

Em C, o desenvolvedor precisa gerenciar manualmente a memória por meio de funções como malloc() e free(), além de manter arquivos de cabeçalho (.h) separados dos arquivos de implementação (.c). O uso frequente de ponteiros e callbacks também aumenta a complexidade do código.

Go

Go utiliza coleta automática de lixo (Garbage Collector), dispensa arquivos de cabeçalho e evita ponteiros aritméticos. Essas características contribuem para um código mais legível, reduzindo a quantidade de linhas necessárias e facilitando a manutenção.

Confiabilidade

Linguagem C

A linguagem C não oferece mecanismos nativos para detecção de condições de corrida (race conditions). Erros de sincronização podem resultar em falhas de segmentação ou corrupção silenciosa de dados.

Go

Go disponibiliza primitivas de sincronização seguras e a ferramenta -race, capaz de detectar condições de corrida durante a execução do programa. Isso contribui para o desenvolvimento de aplicações concorrentes mais robustas e confiáveis.

3. Estrutura Geral do Código

O código está organizado em diretórios modulares, cada um com uma responsabilidade específica:

  • cmd/: Contém os pontos de entrada (arquivos main.go) para iniciar o Tracker e o Peer.
  • peer/: Implementa toda a lógica de cliente e servidor do peer, incluindo o gerenciamento de arquivos compartilhados e o motor de download paralelo.
  • tracker/: Contém a lógica do servidor central que mantém o índice de quais peers estão online e quais arquivos cada um possui.
  • protocol/: Define o protocolo de comunicação baseado em JSON e o sistema de enquadramento (framing) para envio de dados binários (chunks).
  • web/: Implementa a interface gráfica simplificada utilizando o servidor HTTP nativo do Go.

4. Principais Funções

Abaixo estão as funções essenciais que movem o sistema:

  • DownloadFile() (Peer): Orquestra o download paralelo, dividindo o arquivo em pedaços e distribuindo a tarefa entre os workers disponíveis.
  • serverLoop() (Peer): Loop infinito que aceita novas conexões de outros peers e dispara uma goroutine (handlePeerRequest) para cada uma.
  • Start() (Tracker): Inicia o servidor central e aguarda conexões persistentes dos peers para registro de arquivos.
  • AddSharedFile() (Peer): Lê os metadados de um arquivo local e o anuncia para o Tracker, tornando-o disponível para a rede.
  • SearchFile() (Peer): Consulta o Tracker para obter uma lista de quais peers possuem um determinado arquivo.
